# Governance

Habilis currently uses a maintainer-led governance model.

## Roles

### Maintainer

Maintainers are responsible for:

- Setting project direction and roadmap priorities
- Reviewing and merging pull requests
- Managing releases
- Enforcing the Code of Conduct
- Handling security reports and coordinating fixes

### Contributor

Contributors are anyone who helps improve the project through code, docs,
design, testing, issue triage, or feedback.

## Decision Making

- Day-to-day decisions happen in issues and pull requests
- Significant architecture decisions should be recorded in `docs/decisions.md`
- Maintainers make the final merge decision when consensus is unclear
- Project health, safety, and long-term maintainability take priority over short-term convenience

## Releases

Releases are cut by maintainers. The project may delay merging a change until it
has adequate documentation, tests, or operational clarity.

## Governance Changes

Changes to this governance model can be proposed through a pull request and take
effect once approved by the maintainers.
