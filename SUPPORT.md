# Support

Use the channel that best matches what you need:

- Bug reports: open a GitHub issue using the bug report template
- Feature requests: open a GitHub issue using the feature request template
- Pull requests: open a PR when you already have a concrete fix or improvement
- Security issues: follow [`SECURITY.md`](SECURITY.md) and report privately

## Getting Good Help

When asking for help, include:

- What you were trying to do
- What you expected to happen
- What happened instead
- Relevant logs, screenshots, or request/response samples
- Your local environment details when setup is involved

## Response Expectations

Support is best-effort while the project is still maintainer-led. Clear, reproducible reports will get the fastest response.
